# VELA 컨트랙트 공정성 및 보안 백서

> 본 문서는 모든 VELA 커뮤니티 사용자를 대상으로 하며, VELA 스마트 컨트랙트의 보안성, 공정성 및 투명성에 대한 설계와 보장을 종합적으로 설명하는 것을 목적으로 합니다.

---

## 1. 제3자 권위 감사

VELA의 스마트 컨트랙트는 세계 1위 블록체인 보안 감사 기관인 **CertiK**의 엄격한 감사를 통과했습니다. CertiK은 Aave, Polygon, BNB Chain 등 유명 프로젝트를 포함하여 4,000개 이상의 블록체인 프로젝트에 보안 감사 서비스를 제공해 왔으며, 업계에서 가장 권위 있는 보안 감사 기관으로 인정받고 있습니다.

감사 내용:
- 컨트랙트 코드 로직의 완전성 검토
- 일반적인 취약점 스캔 (재진입 공격, 오버플로우, 권한 우회 등)
- 백도어 탐지 및 자금 보안 검증
- 자금 흐름 로직의 안전성 및 신뢰성 확인

감사 결론: 컨트랙트 코드에 보안 취약점이 없으며, 어떠한 백도어나 숨겨진 관리자 출금 기능도 존재하지 않습니다.

감사 보고서 공개 조회: **https://skynet.certik.com/projects/vela**

누구나 CertiK 공식 웹사이트에서 VELA의 보안 점수와 감사 세부 사항을 확인할 수 있습니다.

## 2. 코드 완전 오픈소스

VELA는 "코드가 곧 법"이라는 신념을 가지고 있으며, 모든 코드를 완전히 공개하여 사회 전체의 감독을 받고 있습니다:

**스마트 컨트랙트 소스 코드**
- 컨트랙트 코드는 GitHub에 호스팅되어 있으며, 누구나 조회 및 검토 가능
- 저장소 주소: https://github.com/veladex/vela-anchor

**프론트엔드 웹사이트 코드**
- 웹사이트 보안에 대한 사용자의 우려를 해소하기 위해, 패키징된 프론트엔드 코드도 오픈소스로 공개
- 저장소 주소: https://github.com/veladex/vela-website

이는 VELA에 어떠한 "블랙박스"도 없다는 것을 의미합니다 — 온체인 컨트랙트부터 사용하시는 웹 인터페이스까지, 모든 것이 투명합니다.

## 3. 온체인 코드 검증 — 직접 확인하세요

오픈소스 코드가 정말로 온체인에서 실행되는 프로그램과 동일할까요? 저희의 말을 믿을 필요 없이, 직접 검증하실 수 있습니다.

Solana는 공식 코드 검증 도구 `solana-verify`를 제공하며, 누구나 다음 명령어를 실행하여 GitHub의 소스 코드와 온체인에 배포된 프로그램을 바이트 단위로 비교할 수 있습니다:

```bash
solana-verify verify-from-repo \
    --url https://api.mainnet-beta.solana.com \
    --program-id FW6P7G9yPBqGAGsZ6Aa7upC9whF69QMH4ZJaBJjFsLVK \
    https://github.com/veladex/vela-anchor
```

검증 결과:

```
Executable Program Hash from repo: 5a9221595cb0f03287e8f58c1613173316ff036f274668f4db716d8eab2fc343
On-chain Program Hash: 5a9221595cb0f03287e8f58c1613173316ff036f274668f4db716d8eab2fc343
Program hash matches ✅
```

두 해시 값이 완전히 일치하여, 온체인에서 실행되는 프로그램이 오픈소스 코드의 컴파일 결과와 완전히 동일하며 어떠한 변조도 없음을 증명합니다.

---

## 4. 컨트랙트 코드 검증 가능

컨트랙트 배포 후, 코드는 Solana 블록체인에 고정되며 고유한 프로그램 해시가 생성됩니다. 이 해시는 CertiK 감사를 통해 인증되고 공개적으로 기록되었습니다. 이는 다음을 의미합니다:

- 컨트랙트 코드에 대한 어떠한 수정도 완전히 다른 프로그램 해시를 생성하여, CertiK의 감사 검증을 통과할 수 없음
- 동일한 소스 코드로 재배포하더라도 새로운 컨트랙트 프로그램 주소가 생성되어, 감사된 기록과 일치하지 않음
- 커뮤니티의 누구나 `solana-verify` 도구를 사용하여 언제든지 온체인 프로그램과 오픈소스 코드를 비교하여 일관성을 확인할 수 있음

귀하의 모든 스테이킹과 수익 인출은 온체인 코드에 작성된 규칙에 따라 엄격하게 실행되며, CertiK 감사의 지속적인 보장을 받습니다.

## 5. 자금 보안 — 아무도 당신의 돈을 건드릴 수 없습니다

이것은 사용자들이 가장 관심을 가지는 문제이며, 가장 명확한 답변을 드립니다:

**컨트랙트에는 관리자가 사용자 자금을 이체할 수 있는 어떠한 기능도 존재하지 않습니다.**

- 귀하의 자금은 블록체인의 전용 계정(PDA)에 저장되며, 컨트랙트 프로그램에 의해 제어됩니다
- 모든 자금 작업(스테이킹, 상환, 수익 인출)은 귀하 본인의 지갑 서명이 필요합니다
- 관리자의 권한은 초기화 설정(NFT 컬렉션 생성, 추천 시스템 초기화 등)에만 제한되며, 자금 흐름과는 완전히 무관합니다
- 이자 수령 시 공제되는 10% 세금은 Solana의 소각 주소(블랙홀 주소)로 직접 전송되며, 누구의 지갑에도 들어가지 않습니다

VELA 웹사이트가 폐쇄되더라도, 귀하의 자금은 블록체인에 안전하게 존재하며 온체인 상호작용을 통해 언제든지 회수할 수 있습니다. 자세한 자금 회수 방법은 "자금 보안 안내"를 참조하세요.

## 6. 공정하고 투명한 규칙

### 공개 투명한 스테이킹 규칙

| 스테이킹 기간 | 일일 이율 | 스테이킹 범위 |
|---------|--------|---------|
| 7일 | 0.5% | 1,000 - 50,000 VELA |
| 30일 | 0.7% | 1,000 - 50,000 VELA |
| 90일 | 1.0% | 1,000 - 50,000 VELA |

위 이율은 컨트랙트 코드에 하드코딩되어 있으며, 모든 사용자에게 완전히 동일하게 적용됩니다. 어떠한 "특별 대우"나 "숨겨진 규칙"도 존재하지 않습니다.

### 세금 사용처 투명

이자 수령 시 공제되는 10% 세금은 100% Solana 체인의 소각 주소로 전송됩니다. 이 과정은 완전히 온체인에서 실행되며, 누구나 블록 탐색기(Solscan, Solana Explorer 등)를 통해 실시간으로 확인할 수 있습니다.

### NFT 수량 제한 및 공개

- Diamond NFT 최대 공급량: 600개
- Gold NFT 최대 공급량: 12,000개

공급 상한은 컨트랙트 코드에 하드코딩되어 있으며, 추가 발행이 불가능합니다.

## 7. 웹사이트에 의존하지 않음 — 진정한 탈중앙화

VELA는 Solana 블록체인에 완전히 배포된 탈중앙화 프로토콜입니다. 웹사이트는 단지 편리한 조작 인터페이스일 뿐이며, 컨트랙트 자체는 블록체인에서 독립적으로 운영됩니다.

웹사이트를 거치지 않더라도 컨트랙트와 직접 상호작용하여 모든 작업을 완료할 수 있는 오픈소스 상호작용 스크립트를 제공합니다. 스크립트 코드는 프로젝트 저장소의 `scripts/stake-example.js`에 위치하며, 사용 방법은 다음과 같습니다:

1. Node.js 환경 설치 (v16 이상)
2. 코드 저장소 클론 및 의존성 설치
   ```bash
   git clone https://github.com/veladex/vela-anchor.git
   cd vela-anchor/scripts
   npm install
   ```
3. `stake-example.js`에서 정보 설정:
   - `privateKeyString`을 지갑 개인키로 교체 (bs58 형식)
   - `connection`의 RPC 주소를 Solana 메인넷 RPC로 변경 (예: `https://api.mainnet-beta.solana.com`)
   - `mintAddress`를 메인넷 VELA 토큰 주소로 변경
4. 필요에 따라 해당 함수 호출:
   ```bash
   node stake-example.js
   ```

스크립트에는 완전한 작업 예제가 포함되어 있습니다:

| 함수명 | 기능 | 설명 |
|--------|------|------|
| `addReferral` | 추천인 연결 | 스테이킹 전 반드시 추천인을 연결해야 함 |
| `createStake` | 스테이킹 생성 | 금액과 기간 유형 지정 (1=7일, 2=30일, 3=90일) |
| `getMyStakingOrders` | 스테이킹 주문 조회 | 진행 중인 모든 스테이킹 확인 |
| `claimInterest` | 이자 수령 | 특정 주문의 누적 이자 수령 |
| `unstake` | 스테이킹 해제 | 만기 후 원금 및 잔여 이자 상환 |

모든 작업은 본인의 지갑 서명만 필요하며, 제3자 승인은 필요하지 않습니다.

> 참고: 기술 작업에 익숙하지 않은 경우, 커뮤니티의 개발자에게 검증 및 실행을 요청할 수 있습니다 — 코드가 완전히 오픈소스이므로, 누구나 스크립트의 안전성을 확인할 수 있습니다.

## 9. 자주 묻는 질문

**Q: 프로젝트 팀이 자금을 가지고 도주할 수 있나요?**
A: 아닙니다. 컨트랙트에는 관리자 출금 기능이 없으며, 자금은 사용자 본인만이 지갑 서명을 통해 조작할 수 있습니다. 팀이 해산되더라도, 귀하의 자금은 체인에 안전하게 존재하며 언제든지 회수할 수 있습니다.

**Q: 컨트랙트가 몰래 업그레이드되어 백도어가 추가될 수 있나요?**
A: 아닙니다. 컨트랙트 코드는 CertiK 감사를 통과하고 고유한 프로그램 해시를 생성했습니다. 어떠한 코드 수정도 해시를 변경하여 감사 검증을 통과할 수 없으며, 소스 코드로 재배포하더라도 새로운 프로그램 주소가 생성되어 CertiK에서 인정하지 않습니다. `solana-verify` 도구를 사용하여 언제든지 온체인 코드와 오픈소스 저장소의 일관성을 독립적으로 검증할 수 있습니다.

**Q: 이율이 몰래 수정될 수 있나요?**
A: 아닙니다. 이율 매개변수는 컨트랙트 코드에 하드코딩되어 있으며, 수정 가능한 변수가 아닙니다. 이율을 변경하려면 완전히 새로운 컨트랙트(새 주소)를 배포해야 하며, 기존 컨트랙트의 스테이킹은 어떠한 영향도 받지 않습니다.

**Q: 추천 수익 계산이 공정한가요?**
A: 추천 수익의 계산 로직은 온체인 컨트랙트에 완전히 작성되어 있으며, 모든 사람에게 동등하게 적용됩니다. 오픈소스 코드나 감사 보고서를 확인하여 계산 규칙을 검증할 수 있습니다.

**Q: NFT가 무한 발행되어 가치가 희석될 수 있나요?**
A: 아닙니다. Diamond NFT 상한 600개, Gold NFT 상한 12,000개이며, 이 숫자들은 컨트랙트에 하드코딩되어 있어 수정이 불가능합니다.

**Q: 기술을 모르는데, 이러한 주장이 사실인지 어떻게 확인하나요?**
A: Solana 개발을 이해하는 친구에게 검증을 요청하거나, CertiK의 감사 보고서를 참조할 수 있습니다. 저희 코드는 완전히 오픈소스이며, 누구의 검증도 견딜 수 있습니다.

**Q: 컨트랙트가 해킹당할 수 있나요?**
A: VELA 컨트랙트는 세계 1위 CertiK의 전문 감사를 받았으며, Solana 생태계에서 가장 성숙한 Anchor 프레임워크로 개발되어 업계 최고 보안 표준을 준수합니다. 또한 코드가 완전히 오픈소스이며, 전 세계 보안 연구자들의 지속적인 검토를 받고 있습니다.

**Q: 웹사이트가 개인키를 탈취할 수 있나요?**
A: 아닙니다. VELA 컨트랙트와 웹사이트는 귀하의 개인키를 저장하거나 접근하지 않습니다. 모든 거래 서명은 귀하의 지갑(예: Phantom)에서 로컬로 완료됩니다. 프론트엔드 코드도 오픈소스이며, 누구나 확인할 수 있습니다. 개인키나 니모닉 문구를 반드시 안전하게 보관하세요. 이것이 자금 보안의 유일한 보장입니다.
